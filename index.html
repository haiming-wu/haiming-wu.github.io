Label Embedding Network: Learning Label Representation for Soft Training of Deep Networks

Posted on 2021-01-11

最近学习label embedding的相关内容，总结一下北京大学于2017年发表在NIPS的一篇文章。

摘要

本文提出一种label embedding network的方法，该方法通过反向传播自适应地学习标签向量，从而可以在深度网络的训练过程中学习label表示，即label embedding。主要是将原有的one-hot表示的损失函数转化为一个新的软分布损失函数，使得理论上无关的标签在训练过程中具有连续的相互作用。实验结果表面，训练后的模型具有较高的精度和较快的收敛速度。并且给出代码地址：http://github.com/lancopku/LabelEmb

Quantum Language Model with Entanglement Embedding for Question Answering

Posted on 2021-01-10 Edited on 2021-01-11

  • 单位：浙江大学

  • 作者： Yiwei Chen，Yu Pan， Daoyi Dong

摘要

在量子语言模型中，词语被建模为语素的量子叠加态，这具有很好的模型透明度和良好的解释能力。然而，在目前的文献中，词语顺序基本上被建模为经典的词语混合状态（词语的语序信息没能被很好建模出来？），这不能充分利用量子概率描述的潜力。因此，一种能用于明确捕捉单词序列中非经典关联的完整量子语言模型还没被建立。本文提出了一种基于量子纠缠词向量模式的全新神经网络模型，其功能是将多个量子系统的字序列转换成纠缠纯态。强量子纠缠是量子信息的核心概念，是单词间平行关联的一种表示，它在字序内被观察到。数值试验表明，与传统的深度神经网络模型和其它两字语言模型相比，本文提出的基于纠缠词向量的量子语言模型具有更好的性能。另外，通过量化词间纠缠度，可以提高模型的自组织解释能力。

Label Embedding for MTL

Posted on 2020-12-26 Edited on 2021-01-11

Embedding在NLP任务中是一个非常值得关注的点，传统的研究中更多关注word-level embedding和sentence-level embedding，并且计算出最后的分布式表示。不过这些工作在处理

“Multi-Task Label Embedding for Text Classification”

该文章将针对文本分类问题进行研究，通过高质量标注数据将问题转移到MTL，并对现有MTL中存在的三个问题进行总结：

1、缺乏标签信息，无依赖和信息含义的One-hot向量可能会带来潜在标签信息丢失；

2、不可扩展，网络结构是为当前MTL设计的，有的只能用于两个任务，当加入新任务，需要修改网络结构或者重新训练全部网络；

3、不能迁移，许多现有网络结构是固定的，没有为新任务设计可用layer。

Label Embedding 在之前的文献中就被用来增加通用神经文本分类模型。本文用Label Embedding去提升MTL。

方法

输入x，基于预训练的word2vec将x映射为句子矩阵，而后用Bi-LSTM进行编码表示；同时，对所有标签y映射语义向量表示；最后，依次计算句子表示和每个标签的匹配得分s（用一个矩阵M乘以两个表示的拼接），并对所有的得分进行softmax归一化，而后和真实标签计算交叉熵损失。

扩展和迁移：本文重点研究了这两个方面，通过设计了3组实验。
扩展：Hot Update，新任务在原模型数据上进行微调；Cold Update，在新旧所有数据集上重新训练模型。
迁移：Zero Update，直接在原模型上进行新数据的测试。

201009.张亚洲讲座

Posted on 2020-10-09

时间：2020-10-09

讲座人：张亚洲

题目：情感分析的哲学浅见

哲学与情感

哲学：没有固定答案，对真实事件中的一些批判性思考和人生的反思，学到深处即哲学。

人与情感：人是一种总体性的存在，一种具有无限可能的存在。

情感和情绪：情感具有稳定性，情绪有即时性、非恒常性、非稳定性。

情感分析：对情感与逻辑关系的讨论。

逻辑是理性的，而情感是感性的，在近些年，随着心理学、认知科学的发展，存在理性与非理性的情感，而且目前关注的更多是情感逻辑。

而与之相近的理性逻辑则很容易被计算机模仿，但处理情感逻辑则很难了。

理性逻辑 –> 认识世界

情感逻辑 –> 保护自己

情感分析遵守的几条规律：

  • 1.自发性

    情感产生是自动的，无需经过意识活动之中的推理过程，而是潜意识中的。

  • 2.生存法则

    所有的情感都是为了直接或间接地保护人的安全，为生存服务。
    情感往往基于理性基础，起源于对外界的感知，会引导感知并影响情感选择。（现在的情感分析都是基于事理逻辑的）

  • 4.不确定性

    理性逻辑在人类认识“不变”的自然或规律的时候起到作用。然而，一旦面对变化莫测的事物时，理性逻辑就难以发挥作用。

    情感和量子一样，具有“测不准”的特征。

    情感可能发生跳转，对同一事件，可能突然从一种情感跳到另一种情感，而且不符合形式逻辑的理由。

    由此引入量子理论是十分有必要的。

    理性往往可以用相对线性、秩序性、普通性的范畴、公式、原理等标示。情感则更多具有非线性特征，很难用线性、秩序、普通性的范畴、公式、原理来表示。

    情感是有非理性特征的

复值模糊网络（–>面向对话任务）

1.情感表达与理解缺少对模糊性的关注

量子理论与模糊逻辑之间存在联系。

模糊逻辑和量子测量，隶属度函数

量子测量：量子系统与测量是100%交互的

目前，量子系统、量子系统周围的环境和测量系统三者耦合；

模糊测量：量子系统、周围环境和测量系统三者耦合程度各有不同。（测量算子不再是原来那种，waby的工作还是传统的测量）

2.改进

（1）目前是纯文本的，但过度迎合模糊，没有其他模式，后期做到多模态、多任务等方向（目前：情感和封测）

（2）偏见信息或歧视等问题的感知网络，用于对话系统。

3.意见

NLP还缺乏理论和哲学层面的证明。

[统计]特征选择方法

Posted on 2020-07-27 Edited on 2020-08-24

特征选择之信息增益

特征选择

特征选择在于选取队训练数据具有分类能力的特征，是决定用哪个特征来划分特征空间。

通常特征选择的准则是信息增益或信息增益比。

信息增益

熵和条件熵

熵（entropy）是表示随机变量不确定性的度量，设X是一个取有限个值的离散随机变量，起概率分布为：
$$
P(X=x_i) = p_i, i = 1,2,…,n
$$
则随机变量X的熵为：
$$
H(X)=-\sum^n_{i=1}p_i log p_i
$$

Python代码规范与命名规则

Posted on 2020-06-16 Edited on 2020-06-17

主要涉及5类命名规则：模块、类名、函数、变量和常量，为了加深记忆，进一步和Java进行了对比。

1.模块

  • 模块尽量使用小写命名，首字母保持小写，尽量不要用下划线(除非多个单词，且数量不多的情况)

2.类名

  • 类名使用驼峰(CamelCase)命名风格，首字母大写，私有类可用一个下划线开头

3.函数

  • 函数名一律小写，如有多个单词，用下划线隔开
  • 私有函数在函数前加一个下划线_

4.变量

  • 变量名尽量小写, 如有多个单词，用下划线隔开
  • 常量采用全大写，如有多个单词，使用下划线隔开

5.常量

  • 常量使用以下划线分隔的大写命名

6.Python与Java对比

Python Java
模块/包 尽量使用小写命名，首字母保持小写，尽量不要用下划线(除非多个单词，且数量不多的情况) 每个字母都需要小写
类 类名使用驼峰(CamelCase)命名风格，首字母都大写，私有类可用一个下划线开头 首字母都大写，而且中间不使用任何的连接符。尽量使用英文。如ConnectionFactory
函数/方法 1）一律小写，如有多个单词，用下划线隔开
2）私有函数在函数前加一个下划线_		 首单词全部小写，如果一个方法由多个单词构成，那么从第二个单词开始首字母大写，不使用连接符。
变量/属性 1）变量名尽量小写, 如有多个单词，用下划线隔开
2）常量采用全大写，如有多个单词，使用下划线隔开		 与方法相同。如ageOfPerson
